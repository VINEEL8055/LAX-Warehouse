Database Architecture
Supabase PostgreSQL

2 core tables · RLS protected · JSONB support

All data is scoped to authenticated users only via Row Level Security policies

📋
weekly_snapshots
1 row per saved date

Stores the aggregate summary for each saved week — total capacity, occupancy, utilization %, floor pallets, shelf type breakdown, and per-account floor data as JSONB.

PK: week_label JSONB: floor_accounts auto: timestamps
📦
weekly_records
1 row per shelf per saved date

Stores shelf-level detail — each of the 26 shelves gets its own row with front/bridge/back empties, occupied count, free count, and utilization %.

PK: id (serial) FK: week_label UNIQUE: week+shelf
8
RLS Policies
2
Indexes
1
Unique Constraint
All operations (SELECT, INSERT, UPDATE, DELETE) require an active authenticated session. Anonymous access is completely blocked by RLS.

INDEXES

-- Fast lookup by date
CREATE INDEX idx_weekly_records_week
  ON weekly_records(week_label);

-- Prevents duplicate shelf per week
CREATE UNIQUE INDEX idx_weekly_records_week_shelf
  ON weekly_records(week_label, shelf_id);

ON DELETE CASCADE

If a snapshot row is deleted, all its associated shelf records are automatically removed. This keeps the database clean with no orphaned rows.

weekly_snapshots → weekly_records
1 snapshot = up to 26 shelf records

Data Flow
Save → Query → Display

How data moves through the system

SAVE WEEK FLOW

// 1. User enters shelf empties + floor accounts
inputs[shelf_id] = "46-3-25" // F-B-K
floorAccounts = [{name:"Amazon", value:"45"}]

// 2. apply() calculates occupancy
occ = cap − (fE + bE + kE)
util = (occ / cap) × 100

// 3. Upsert aggregate snapshot
supabase.from("weekly_snapshots")
  .upsert(snap, {onConflict: "week_label"})

// 4. Insert shelf-level records
supabase.from("weekly_records")
  .insert(recs) // up to 26 rows

QUERY FLOW (HISTORICAL)

// Load all saved snapshots (trends page)
supabase.from("weekly_snapshots")
  .select("*")
  .order("week_label", {ascending: true})
  .limit(52)

// Load one snapshot for analytics
supabase.from("weekly_snapshots")
  .select("*")
  .eq("week_label", weekLabel)
  .single()

// Load shelf records for report history
supabase.from("weekly_records")
  .select("*")
  .eq("week_label", weekLabel)
  .order("shelf_id", {ascending: true})
